# 🛠️ Disk Fix Script v6.0

**Disk Fix Script** — это мощный инструмент для работы с медленными (проблемными) секторами на жёстких дисках. Скрипт автоматически обнаруживает области с низкой скоростью чтения, блокирует их с помощью разреженных файлов (sparse files) и предотвращает запись данных в проблемные зоны.

## 🔧 Параметры конфигурации

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `DISK` | Диск для сканирования | `/dev/sda` |
| `PARTITION` | Раздел для анализа файлов | `/dev/sda1` |
| `TARGET_MOUNT` | Точка монтирования диска | `/media/ubuntu/...` |
| `MIN_SPEED` | Минимальная скорость (МБ/с). Ниже — сектор считается медленным | `20` |
| `BEFORE_MB` | МБ **до** медленной зоны (защитная зона перед проблемой) | `100` |
| `AFTER_MB` | МБ **после** медленной зоны (защитная зона после проблемы) | `400` |
| `WAIT_SEC` | Время ожидания после проблемного чтения (секунд) | `5` |
| `BLOCK_SIZE_MB` | Размер блока при сканировании (МБ) | `10` |

### 📐 Как рассчитывается защитная зона
- Обнаружен медленный сектор на позиции X МБ
- ↓
- Защитная зона: [X - BEFORE_MB] до [X + AFTER_MB]
- ↓
- Пример: X = 1600 МБ, BEFORE_MB = 100, AFTER_MB = 400
- ↓
- Заглушка: seek = 1500, count = 500 МБ

## 📊 Формат базы данных (slow_zones.txt)
- Простой текстовый формат: seek|count
- Каждая строка = одна защитная зона
- 1500|500 # зона с seek=1500, размером 500 МБ
- 12690|1000 # зона с seek=12690, размером 1000 МБ
- 24250|500 # зона с seek=24250, размером 500 МБ
- 35730|1000 # зона с seek=35730, размером 1000 МБ

**Преимущества формата:**
- ✅ Человекочитаемый, легко редактируется в любом текстовом редакторе
- ✅ Не требует установки парсеров JSON
- ✅ Устойчив к повреждениям (даже при обрыве записи)
- ✅ Простота обработки через `cut`, `grep`, `awk`

⚙️ Технические детали
Sparse Files (разреженные файлы)
Разреженный файл — это файл, который имеет виртуальный размер, но занимает меньше места на диске, так как не хранит нулевые блоки.


# Обычный файл (записывает нули на диск)
dd if=/dev/zero of=file bs=1M count=1000  
# → Медленно (~10-30 секунд), изнашивает диск, занимает 1000 МБ

# Разреженный файл (sparse file)
truncate -s 1000M file                     
# → Мгновенно (0.001 секунды), не изнашивает диск, занимает 0 байт
Сравнение методов создания файлов:

Метод	Время на 1000 МБ	Физическая запись	Износ диска	Реальное место
dd if=/dev/zero	~10-30 сек	Да	Высокий	1000 МБ
fallocate	~0.1 сек	Да (только метаданные)	Низкий	~0 МБ
truncate (sparse)	~0.001 сек	Нет	Нет	0 байт
Атрибут +i (immutable / неизменяемый)
Атрибут +i делает файл неизменяемым — даже суперпользователь не может его удалить или изменить.

bash
# Установка защиты от удаления
chattr +i file

# Попытка удаления
rm file
# Результат: rm: cannot remove 'file': Operation not permitted

# Снятие защиты (только если нужно удалить)
chattr -i file
rm file  # Теперь работает
Почему GParted не видит занятое место?
Sparse-файлы не занимают физического места на диске, поэтому утилиты вроде GParted показывают их как свободное пространство. Однако файловая система знает о резервировании и не будет записывать данные в эти области.

bash
# Виртуальный размер (что видит df и ls)
ls -lh .slow_patch_1500_500MB
# -rw-r--r-- 1 root root 500M May 24 10:00 .slow_patch_1500_500MB

# Реальный размер на диске (что видит du)
du -sh .slow_patch_1500_500MB
# 0       .slow_patch_1500_500MB
⚠️ Важные предупреждения
Предупреждение	Подробное описание
Потеря данных	Существующие файлы, находящиеся в медленных зонах, будут ПЕРЕЗАПИСАНЫ при создании заглушек. Невозможно восстановить!
Обязательный бэкап	Перед использованием скрипта настоятельно рекомендуется сделать полную резервную копию важных данных
Защита файлов	Созданные заглушки защищены атрибутом +i. Для их удаления нужно сначала снять защиту: sudo chattr -i .slow_patch_*
Права root	Скрипт работает с дисками на низком уровне и требует прав суперпользователя. Запускайте только через sudo
Не для SSD с TRIM	На SSD дисках медленные сектора обычно означают физическую смерть ячеек. Рекомендуется замена диска
🐛 Устранение неполадок
Проблема	Решение
GParted не видит занятое место	Sparse-файлы занимают 0 байт физически. Используйте df -h для виртуального размера, du -sh для реального.
Не удаляется файл заглушки	Снимите защиту: sudo chattr -i .slow_patch_*, затем удаляйте.
Ошибка "bc: command not found"	Установите bc: sudo apt install bc (Debian/Ubuntu) или sudo yum install bc (CentOS/RHEL).
Диск не обнаружен	Проверьте, что диск подключён: lsblk и fdisk -l. Убедитесь, что указан правильный путь.
Сканирование очень медленное	Увеличьте BLOCK_SIZE_MB или уменьшите MIN_SPEED.
Много ложных срабатываний	Уменьшите MIN_SPEED или увеличьте BEFORE_MB и AFTER_MB.
🪟 Работа с дисками ext4 в Windows через WSL
Что такое WSL?
WSL (Windows Subsystem for Linux) — это среда совместимости от Microsoft, позволяющая запускать нативные Linux-приложения непосредственно в Windows. С помощью WSL вы можете запускать этот скрипт на Windows-компьютере и полноценно работать с дисками, отформатированными в ext4.

Установка WSL
powershell
# Запустите PowerShell от имени администратора

# Установка WSL с Ubuntu по умолчанию
wsl --install

# После установки перезагрузите компьютер

# Установка конкретной версии Ubuntu (опционально)
wsl --install -d Ubuntu-22.04
Подключение физического диска к WSL
Способ 1: wsl --mount (Windows 11, рекомендуется)
powershell
# 1. В PowerShell (администратор) определите номер диска
wmic diskdrive list brief

# Результат:
# Caption                    DeviceID            Size
# Samsung SSD 850 EVO        \\.\PHYSICALDRIVE0  250059350016
# WDC WD10EZEX               \\.\PHYSICALDRIVE1  1000204886016

# 2. Примонтируйте диск (например, диск 1)
wsl --mount \\.\PHYSICALDRIVE1

# 3. Или примонтируйте конкретный раздел
wsl --mount \\.\PHYSICALDRIVE1 --partition 1

# 4. Проверьте в WSL
wsl
lsblk
# Должен появиться /dev/sdc или /dev/sdd
Способ 2: Через USB/IP (для USB-дисков)
powershell
# 1. Установите usbipd-win
winget install --interactive --exact dorssel.usbipd-win

# 2. Перезагрузите компьютер

# 3. В PowerShell (администратор) привяжите USB-устройство
usbipd wsl list
usbipd wsl attach --busid <BUSID>

# 4. В WSL устройство появится как /dev/sdX
Монтирование ext4 раздела в WSL
bash
# 1. Зайдите в WSL
wsl

# 2. Посмотрите доступные диски
lsblk

# Пример вывода:
# NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINTS
# sdc      8:32   0 931.5G  0 disk
# └─sdc1   8:33   0 931.5G  0 part

# 3. Создайте точку монтирования
sudo mkdir -p /mnt/ext4_disk

# 4. Примонтируйте раздел (замените sdc1 на ваш раздел)
sudo mount -t ext4 /dev/sdc1 /mnt/ext4_disk

# 5. Проверьте, что диск примонтирован
df -h | grep ext4_disk
# /dev/sdc1       915G  200G  715G  22% /mnt/ext4_disk
Запуск скрипта для работы с ext4 диском
bash
# 1. Перейдите в примонтированный диск
cd /mnt/ext4_disk

# 2. Скопируйте скрипт на диск (если его там нет)
cp /путь/к/disk_fix.sh .

# 3. Сделайте исполняемым
chmod +x disk_fix.sh

# 4. Запустите скрипт
sudo ./disk_fix.sh

# 5. При настройке укажите:
#    Диск: /dev/sdc (или какой определился)
#    Раздел: /dev/sdc1
#    Точка монтирования: /mnt/ext4_disk
Доступ к файлам из Windows
После монтирования диска в WSL, все файлы становятся доступны из Windows через специальный сетевой путь:

text
\\wsl.localhost\Ubuntu\mnt\ext4_disk
Как открыть:

Откройте Проводник Windows (Win + E)

В адресной строке введите: \\wsl.localhost\Ubuntu\mnt\ext4_disk

Нажмите Enter

Теперь вы можете:

📂 Просматривать файлы как обычные папки

📋 Копировать файлы через Ctrl+C / Ctrl+V

✏️ Редактировать файлы в любом Windows-редакторе (Notepad++, VS Code и др.)

📁 Создавать, переименовывать, удалять файлы и папки

Полный скрипт автоматизации для WSL
Создайте файл wsl_mount_and_fix.bat:

batch
@echo off
echo === WSL Disk Fix Helper ===
echo.

set /p DISK_NUM="Введите номер диска (например 1 для PHYSICALDRIVE1): "

if "%DISK_NUM%"=="" (
    echo Отмена
    exit /b
)

echo Монтирование диска PHYSICALDRIVE%DISK_NUM%...
wsl --mount \\.\PHYSICALDRIVE%DISK_NUM% --bare

timeout /t 3 /nobreak >nul

wsl bash -c "
    NEW_DEVICE=\$(lsblk -l | grep -E '^sd[a-z]' | tail -1 | awk '{print \"/dev/\"\$1}')
    echo \"Обнаружено устройство: \$NEW_DEVICE\"
    
    MOUNT_POINT=\"/mnt/ext4_disk_\${NEW_DEVICE##*/}\"
    sudo mkdir -p \"\$MOUNT_POINT\"
    sudo mount -t ext4 \${NEW_DEVICE}1 \"\$MOUNT_POINT\"
    
    echo \"Диск примонтирован в: \$MOUNT_POINT\"
    echo \"Доступ в Windows: \\\\wsl.localhost\\Ubuntu\$MOUNT_POINT\"
    
    cd \"\$MOUNT_POINT\"
    
    if [ ! -f disk_fix.sh ]; then
        echo \"Копирование скрипта...\"
        cp /home/*/disk_fix.sh . 2>/dev/null || wget https://raw.githubusercontent.com/yourrepo/disk-fix/main/disk_fix.sh
        chmod +x disk_fix.sh
    fi
    
    sudo ./disk_fix.sh
    
    read -p \"Отмонтировать диск? (y/n): \" UNMOUNT
    if [[ \"\$UNMOUNT\" =~ ^[Yy] ]]; then
        sudo umount \"\$MOUNT_POINT\"
        echo \"Диск отмонтирован\"
    fi
"

pause
